# ALA-frontend develop branch

<strong>develop</strong>

- main에서 분기
- 기능 개발이 완료된 코드를 관리하는 브랜치
- feature에서 기능 개발이 완료되는 코드를 merge
- CI/CD가 이루어질 브랜치

`commit rule` <br/>

<strong>Feature</strong>

- 새로운 기능 추가

<strong>Fix</strong>

- 버그 수정

<strong>Docs</strong>

- 문서 수정

<strong>Style</strong>

- 코드 포맷팅, 세미콜론 등 코드 변경이 없는 경우

<strong>Refactor</strong>

- 코드 리팩토링

<strong>Chore</strong>

- 그 외, 설정 파일 수정 등

(ex) Feature: 제목 \n 본문 \n #이슈번호
